# Telegram Shop Bot with Strapi CMS

Данный проект представляет собой Telegram-бота для интернет-магазина, интегрированного с CMS Strapi. Бот позволяет просматривать товары, добавлять их в корзину, удалять из корзины, оформлять заказ (сохраняет email покупателя) и просматривать содержимое корзины.

## Технологии

- **Python 3.13+**
- **aiogram 3.x** — фреймворк для Telegram Bot API
- **Strapi 5** — headless CMS
- **requests** — HTTP-запросы к Strapi API
- **environs** — работа с переменными окружения
- **uv** — быстрый менеджер пакетов и виртуальных окружений

## Установка и запуск

1. Клонируйте репозиторий

```bash
git clone <url-репозитория>
cd tg_shop_bot_with_cms
```

2. Установка зависимостей

Проект использует uv для управления зависимостями. Если у вас ещё не установлен uv, выполните:

```bash
pip install uv
```

Создайте виртуальное окружение и установите зависимости:

```bash
uv venv
source .venv/bin/activate         # для Linux/macOS
.\.venv\Scripts\Activate.ps1      # для Windows в powershell
uv sync
```

3. Настройка переменных окружения
   Создайте файл `.env` в корне проекта:

```env
TG_TOKEN=ваш_токен_бота_от_BotFather
ADMIN_CHAT_ID=ваш_telegram_id (для получения логов ошибок)
```

4. Установка и запуск Strapi
   Бот работает с CMS Strapi. Убедитесь, что у вас установлен Node.js (версия 18, 20 или 22). Затем создайте новый проект Strapi:

```bash
npx create-strapi@latest strapi-cms
cd strapi-cms
npm run develop
```

_После запуска откройте http://localhost:1337/admin и создайте администратора._

**Необходимые модели данных**

В Strapi создайте следующие коллекции:

- **Product** (товары):
  - `title` (текст)
  - `description` (текст, длинное)
  - `price` (число, десятичное)
  - `picture` (медиа, одиночное изображение)

- **Cart** (корзина пользователя):
  - `userId` (текст) — Telegram ID пользователя
  - `cart_items` (отношение Cart has many CartItems)

- **CartItem** (элемент корзины):
  - `quantity` (число, дробное) — количество (килограммы)
  - `product` (отношение CartItem belongs to one Product)
  - `cart` (отношение CartItem belongs to one Cart)

- **Customer** (клиент для email):
  - `userId` (текст) — Telegram ID
  - `email` (email)

После создания моделей не забудьте в панели администратора настроить права доступа для роли **Public**:

- для **Product:** `find`, `findOne`
- для **Cart:** `find`, `findOne`, `create`, `update`
- для **CartItem:** `find`, `findOne`, `create`, `update`, del`ete
- для **Customer:** `create`

5. Запуск бота

Убедитесь, что Strapi запущен на http://localhost:1337. Затем выполните:

```bash
python -m bot_tg.tg_bot
```

_Бот начнёт polling и будет отвечать на команду /start._

## Использование

- /start — главное меню со списком товаров и кнопкой «Моя корзина».
- Нажатие на товар — просмотр детальной информации (описание, цена, изображение).
- Под карточкой товара доступны кнопки:
  - «Добавить в корзину» — добавляет 1 кг товара в корзину.
  - «Моя корзина» — переход к корзине.
  - «Назад к списку товаров» — возврат к списку.
- В корзине отображается список добавленных товаров с количеством и стоимостью, а также итоговая сумма.
- Кнопка «Оплатить» — запрашивает email пользователя и сохраняет его в Strapi (модель Customer).
- Кнопка «Назад к списку товаров» — возврат в меню.


## Примечания

- Бот использует синхронные HTTP-запросы к Strapi через библиотеку `requests`. При необходимости можно заменить на асинхронный `aiohttp`.
- Все ошибки, связанные с работой Strapi, логируются и могут отправляться в указанный Telegram-чат (если настроен `ADMIN_CHAT_ID`).
- Для корректной работы убедитесь, что Strapi запущен до старта бота, а в настройках роли Public разрешены необходимые методы.
